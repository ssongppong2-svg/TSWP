# 개발 원칙

• 기능을 추가하기 전에 이유를 생각한다.

• 기능을 삭제하는 것을 두려워하지 않는다.

• 재미없는 기능은 과감히 제거한다.

• 테스트 결과를 최우선으로 반영한다.

• 플레이어의 경험을 중심으로 판단한다.
