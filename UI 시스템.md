# 목적 (Purpose)

본 문서는 TSWP의 모든 UI와 HUD 시스템을 정의한다.

UI는 플레이어가 필요한 정보를 빠르게 확인할 수 있도록
직관적이고 간결하게 설계한다.

---

# 설계 철학 (Design Philosophy)

① 필요한 정보만 보여준다.

② 전투를 방해하지 않는다.

③ 2초 안에 이해할 수 있어야 한다.

④ 스트리머와 시청자가 보기 쉬워야 한다.

⑤ 모든 UI는 설정에서 크기와 투명도를 조절할 수 있다.

---

# HUD

플레이 중 항상 표시된다.

표시 요소

• 체력

• 직업 아이콘

• 장착 아이템

• 스킬 쿨타임

• 부활 횟수

• 현재 방

• 미니맵

• 핑

• 음성 표시

---

# 플레이어 정보

각 플레이어는

• 닉네임

• 체력바

• 직업 아이콘

을 머리 위에 표시한다.

체력이 낮으면

체력바 색상이 변경된다.

---

# 파티 정보

화면 좌측에 표시한다.

표시 내용

• 플레이어 이름

• 체력

• 직업

• 사망 여부

• 음성 채팅 중 여부

---

# 스킬 UI

스킬 아이콘과

남은 쿨타임을 표시한다.

사용 불가능 시

회색으로 표시한다.

---

# 아이템 UI

장착한 5개의 아이템을 표시한다.

마우스를 올리면

효과를 확인할 수 있다.

---

# 부활 UI

남은 공유 부활 횟수를 표시한다.

횟수가 적어질수록

색상이 변경된다.

---

# 미니맵

현재 위치

팀원 위치

목표 방향

특수 오브젝트

를 표시한다.

숨겨진 방은

발견 전까지 표시되지 않는다.

---

# 보스 UI

보스전에서는

화면 상단에

보스 이름

체력

분노 상태

를 표시한다.

기믹 진행 상황이 있을 경우

별도의 게이지를 표시한다.

---

# 상호작용 UI

상호작용 가능한 오브젝트 근처에서

사용 키와 설명을 표시한다.

예시

[E] 문 열기

[E] 레버 당기기

[E] 아이템 줍기

---

# 핑 UI

핑을 사용하면

맵과 화면에 표시된다.

종류

• 이동

• 위험

• 아이템

• 집합

• 도움

---

# 음성 채팅 UI

말하는 플레이어는

닉네임 옆에

마이크 아이콘이 표시된다.

음소거된 플레이어는

별도 아이콘으로 표시한다.

---

# 이모트

플레이어는

언제든 이모트를 사용할 수 있다.

이모트는

캐릭터 머리 위에 표시된다.

---

# 알림

화면 우측 상단에 표시된다.

예시

• 업적 달성

• 아이템 획득

• 보스 등장

• 플레이어 사망

• 플레이어 부활

---

# 결과 화면

게임 종료 후 표시된다.

내용

• MVP

• 각종 통계

• 획득 업적

• 획득 칭호

• 플레이 시간

• 처치한 보스

---

# 설정

플레이어는

다음 UI를 변경할 수 있다.

• UI 크기

• UI 투명도

• HUD 켜기/끄기

• 데미지 숫자 표시

• 미니맵 표시

• 핑 표시

• 음성 채팅 표시

---

# 접근성

지원 기능

• 색약 모드

• 자막

• UI 확대

• 화면 흔들림 감소

• 플래시 효과 감소

---

# 개발 원칙 연결

• 필요한 정보만 제공한다.

• 전투를 방해하지 않는다.

• 스트리머가 보기 쉬운 화면을 만든다.

---

# 테스트 체크리스트

□ UI가 화면을 너무 가리지 않는가

□ 정보가 한눈에 들어오는가

□ 초보자도 쉽게 이해할 수 있는가

□ 스트리머 화면에서도 보기 쉬운가

□ 접근성 기능이 정상 동작하는가
